사용자 레벨 관리 기능 추가

varchar 로 레벨 (등급)을 관리하기에는 용량, 성능 면에서 별로일 수 있음
ENUM 타입을 사용해서 레벨을 코드화하고, 각 숫자에 매핑하며, DB에는 숫자만 저장한다.


DB 수정
- SQL 잘 못 작성할 경우, 즉, update 문에 where 을 작성하지 않을 경우 
- 2 가지 방법으로 검증
  - 쿼리 실행 후 반환 값 (업데이트 row 수) 로 비교
  - 수정하지 않은 객체를 두고 비교


비지니스 로직은 DAO에 어울리지 않음 (DB 접근 기술 책임만 짐)
따라서 Service 단을 만들어 비지니스 로직을 담당


첫 가입시 기본 레벨 BASIC


객체지향적인 코드
- 다른 object의 데이터를 가져와서 작업하는 대신 데이터를 갖고 있는 다른 object에게 작업을 해달라고 요청 한다.
- object에게 데이터를 요구하지 말고 작업을 요청하는 것이 객체지향 프로그래밍의 가장 기본이 되는 원리이다.


트랜잭션 경계설정
- 오토커밋을 끄고, 트랜잭션의 시작을 선언, commit, rollback으로 트랜잭션을 종료하는 작업
- 하나의 connection이 만들어지고 닫히는 범위 안에 존재한다.
- 하나의 DB 커넥션 안에서 만들어지는 트랜잭션 : 로컬 트랜잭션
