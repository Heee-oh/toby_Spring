사용자 레벨 관리 기능 추가

varchar 로 레벨 (등급)을 관리하기에는 용량, 성능 면에서 별로일 수 있음
ENUM 타입을 사용해서 레벨을 코드화하고, 각 숫자에 매핑하며, DB에는 숫자만 저장한다.


DB 수정
- SQL 잘 못 작성할 경우, 즉, update 문에 where 을 작성하지 않을 경우 
- 2 가지 방법으로 검증
  - 쿼리 실행 후 반환 값 (업데이트 row 수) 로 비교
  - 수정하지 않은 객체를 두고 비교


비지니스 로직은 DAO에 어울리지 않음 (DB 접근 기술 책임만 짐)
따라서 Service 단을 만들어 비지니스 로직을 담당


첫 가입시 기본 레벨 BASIC


객체지향적인 코드
- 다른 object의 데이터를 가져와서 작업하는 대신 데이터를 갖고 있는 다른 object에게 작업을 해달라고 요청 한다.
- object에게 데이터를 요구하지 말고 작업을 요청하는 것이 객체지향 프로그래밍의 가장 기본이 되는 원리이다.



## 단일 책임 원칙
하나의 모듈은 한 가지 책임을 가져야한다. 즉, 변경의 이유가 하나여야함
단일 책임을 해주는 아주 핵심 도구가 바로 스프링 DI
인터페이스 도입 - DI 연결 : 단일 책임 원칙, OCP, 결합도 낮춤, 응집도 향상
여러 디자인 패턴에서 자연스럽게 적용

#### 장점
- 변경필요시 수정 대상이 명확

테스트 대역 : 테스트 대상이 되는 오브젝트의 기능만 충실하게 수행하면서 빠르게, 자주 테스트를 실행
가능하도록 하는 것들

# 배운점
잘완성된 클래스는 항상 추상화가 되어있고
사용자가 언제든지 구현해서 갈아 끼울 수 있다. 
이것이 객체 지향 프로그래밍의 핵심이자 스프링 DI이다.
