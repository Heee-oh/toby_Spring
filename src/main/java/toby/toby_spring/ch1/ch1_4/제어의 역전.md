## 팩토리
- 객체의 생성방법을 결정하고 만들어진 오브젝트(객체)를 반환
- 추상 팩토리 패턴, 팩토리 메서드 패턴과는 다른 것이다.


## IoC
- 제어의 역전
  - 프로그램의 흐름을 직접 제어하는 것이 아닌 외부에서 관리하는 것
  - 사용할 객체를 직접 생성하지 않고, 객체의 생명주기를 외부에 위임 
    - 애플리케이션 코드는 프레임워크가 짜놓은 틀에서 수동적으로 동작 (주입받는대로 사용)

- 프레임 워크
  - 프레임워크가 개발자가 만든 애플리케이션 코드를 사용하는 방식
- 라이브러리
  - 라이브러리를 사용하는 애플리케이션 코드는 애플리케이션 흐름을 직접 제어

- 관심, 책임 분리, 유연하게 확장 가능한 구조를 만드는 과정 자체가 IoC를 적용하는 작업
- 스프링은 IoC를 모든 기능의 기초가 되는 기반 기술로 삼고, IoC를 극한까지 적용한 프레임워크이다.
