// 중복 코드를 뽑아내어 메소드로 만드는 것을 리팩토링에서는 '메소드 추출 기법'


## 템플릿 메서드 패턴
- 슈퍼 클래스에 기본적인 로직 흐름 만들기
- 그 기능의 일부를 추상 메서드 or 오버라이딩 가능한 protected 메서드 등으로 만들기
- 서브 클래스에서 필요에 맞게 구현
- 이 예제에서는 getConnection 추상 메서드다.

## 팩토리 메서드 패턴
- 서브클래스에서 구체적인 오브젝트 생성 방법을 결정
- getConnection이 서브 클래스에서 Connection 클래스의 오브젝트를 어떻게 생성할지 결정하므로 팩토리 메서드 패턴이 맞음
- 보통 인터페이스 타입으로 오브젝트를 리턴
- 서브 클래스에서 오브젝트 생성 방법과 클래스를 결정할 수 있도록 미리 정의해둔 메서드
